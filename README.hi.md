क्लाउड कोड के लिए फ्लाइट कंप्यूटर: लॉग रोटेशन, वॉचडॉग, क्रैश बंडल और एमसीपी (MCP) की स्व-जागरूकता।

क्लाउड गार्डियन एक स्थानीय विश्वसनीयता परत है जो क्लाउड कोड सत्रों को सुचारू रूप से चलाने में मदद करती है। यह लॉग की अत्यधिक मात्रा, डिस्क पर दबाव और सिस्टम के अटकने जैसी समस्याओं का पता लगाने से पहले ही उनका निवारण करती है, और जब कोई समस्या होती है तो उसका रिकॉर्ड रखती है। इसके अलावा, यह एक एमसीपी सर्वर प्रदान करती है ताकि क्लाउड स्वयं सत्र के दौरान अपनी निगरानी कर सके।

## यह क्या करता है।

| आदेश। | उद्देश्य। |
|---------|---------|
| `preflight` | क्लाउड प्रोजेक्ट लॉग्स की जांच करें, बड़ी डाइरेक्टरी और फाइलों की रिपोर्ट तैयार करें, और आवश्यकतानुसार स्वचालित रूप से उन्हें ठीक करने का विकल्प चुनें। |
| `doctor` | एक डायग्नोस्टिक बंडल (ज़िप फ़ाइल) बनाएं जिसमें सिस्टम की जानकारी, लॉग फ़ाइलें और जर्नल शामिल हों। |
| `run -- <cmd>` | किसी भी कमांड को "वॉचडॉग" निगरानी के साथ शुरू करें, और यदि कोई त्रुटि या रुकावट आती है, तो स्वचालित रूप से सभी संबंधित फ़ाइलों को एक साथ समेकित करें। |
| `status` | एक बार में स्वास्थ्य जांच: डिस्क की खाली जगह, लॉग फ़ाइलों का आकार, और चेतावनियाँ। |
| `watch` | पृष्ठभूमि प्रक्रिया (बैकग्राउंड डेमॉन): निरंतर निगरानी, घटना ट्रैकिंग, बजट का अनुपालन। |
| `budget` | एक साथ होने वाले कार्यों की सीमा (बजट) को देखें और प्रबंधित करें (दिखाएं/प्राप्त करें/मुक्त करें)। |
| `mcp` | क्लाउड कोड के स्व-निगरानी के लिए, एमसीपी सर्वर (8 उपकरण) शुरू करें। |

## स्थापित करें।

```bash
npm install -g claude-guardian
```

या सीधे चलाएं:

```bash
npx claude-guardian preflight
```

## शुरुआत कैसे करें।

### अपने परिवेश की जांच करें।

```bash
claude-guardian status
```

```

Disk free: 607.13GB [OK]
Claude projects: C:\Users\you\.claude\projects
Total size: 1057.14MB

  my-project: 1020.41MB

Issues found:
  [WARNING] Project log dir is 1020.41MB (limit: 200MB)
  [WARNING] File is 33.85MB (limit: 25MB)

[guardian] disk=607.13GB | logs=1057.14MB | issues=2
```

### स्वचालित रूप से लॉग फ़ाइलों का आकार कम करें।

```bash
claude-guardian preflight --fix
```

यह उपकरण पुराने लॉग फ़ाइलों (gzip प्रारूप में) को व्यवस्थित करता है, और `.jsonl` या `.log` जैसी बड़ी फ़ाइलों को उनकी अंतिम 'एन' पंक्तियों तक छोटा कर देता है। प्रत्येक कार्रवाई को एक लॉग फ़ाइल में दर्ज किया जाता है, ताकि उसकी जानकारी आसानी से प्राप्त की जा सके।

### एक दुर्घटना रिपोर्ट तैयार करें।

```bash
claude-guardian doctor --out ./bundle.zip
```

एक ज़िप फ़ाइल बनाता है जिसमें शामिल हैं:
- `summary.json` — सिस्टम की जानकारी, फ़ाइल आकार की रिपोर्ट, प्रारंभिक जांच के परिणाम।
- `log-tails/` — प्रत्येक लॉग फ़ाइल की अंतिम 500 पंक्तियाँ।
- `journal.jsonl` — अभिभावक (guardian) द्वारा की गई हर गतिविधि का रिकॉर्ड।

### "वॉचडॉग" के साथ दौड़ें।

```bash
claude-guardian run -- claude
claude-guardian run --auto-restart --hang-timeout 120 -- node server.js
```

वॉचडॉग:
1. यह आपके कमांड को एक चाइल्ड प्रोसेस के रूप में शुरू करता है।
2. यह `stdout/stderr` पर गतिविधि की निगरानी करता है।
3. यदि `--hang-timeout` सेकंड तक कोई गतिविधि नहीं होती है, तो यह एक "डॉक्टर बंडल" कैप्चर करता है।
4. यदि प्रोसेस क्रैश हो जाता है, तो यह एक बंडल कैप्चर करता है, और वैकल्पिक रूप से, यह "बैकऑफ़" के साथ पुनः आरंभ हो सकता है।

## एमसीपी सर्वर (वास्तविक अनलॉक)।

"गार्डियन" को एक स्थानीय एमसीपी सर्वर के रूप में पंजीकृत करें ताकि क्लाउड स्वयं अपनी निगरानी कर सके।"

`~/.claude.json` फ़ाइल में निम्नलिखित जानकारी जोड़ें:

```json
{
  "mcpServers": {
    "guardian": {
      "command": "npx",
      "args": ["claude-guardian", "mcp"]
    }
  }
}
```

फिर क्लाउड कॉल कर सकता है:

| उपकरण। | यह क्या परिणाम देता है। |
|------|----------------|
| `guardian_status` | डिस्क, लॉग, प्रक्रियाएं, अटकने का खतरा, बजट, ध्यान का स्तर। |
| `guardian_preflight_fix` | यह प्रक्रिया लॉग फ़ाइलों को नियमित रूप से अपडेट करती है/अनावश्यक डेटा हटाती है, और इसके पहले और बाद की स्थिति की रिपोर्ट प्रदान करती है। |
| `guardian_doctor` | यह एक डायग्नोस्टिक पैकेज (ज़िप फ़ाइल) बनाता है और फ़ाइल का पथ (location) और सारांश (summary) वापस करता है। |
| `guardian_nudge` | सुरक्षित स्वचालित सुधार: यदि लॉग फ़ाइलें बहुत बड़ी हो गई हैं, तो उन्हें ठीक करें, और यदि आवश्यक हो तो बंडल (bundle) को कैप्चर करें। |
| `guardian_budget_get` | वर्तमान में अनुमत समवर्ती उपयोगकर्ताओं की संख्या, उपयोग में मौजूद स्लॉट, और सक्रिय लीज़। |
| `guardian_budget_acquire` | "एक साथ काम करने के लिए स्लॉट का अनुरोध करें (यह लीज़ आईडी वापस करेगा)।" |
| `guardian_budget_release` | जब आप भारी काम पूरा कर लें, तो पट्टा (लीज) रद्द कर दें। |
| `guardian_recovery_plan` | एक चरणबद्ध पुनर्प्राप्ति योजना, जिसमें विशिष्ट उपकरणों के नाम बताए गए हैं जिनका उपयोग किया जाना है। |

यह क्लाउड को यह कहने की अनुमति देता है: "ध्यान दें: यह एक चेतावनी है। `guardian_nudge` प्रक्रिया शुरू की जा रही है, जिसके बाद समवर्तीता (concurrency) को कम किया जाएगा।"

## कॉन्फ़िगरेशन।

तीन नॉब (अन्य सभी सेटिंग्स पूर्वनिर्धारित और उचित मानों के साथ हार्डकोडेड हैं):

| ध्वज। | डिफ़ॉल्ट। | विवरण। |
|------|---------|-------------|
| `--max-log-mb` | `200` | मैक्स प्रोजेक्ट लॉग डायरेक्टरी का अधिकतम आकार (मेगाबाइट में)। |
| `--hang-timeout` | `300` | किसी भी गतिविधि की अनुपस्थिति में, सिस्टम द्वारा "हैंग" घोषित करने से पहले लगने वाला समय। |
| `--auto-restart` | `false` | दुर्घटना या अटकने की स्थिति में स्वचालित रूप से पुनः चालू होना। |

एक अतिरिक्त, पूर्वनिर्धारित सुरक्षा उपाय:
- **डिस्क में 5 जीबी से कम खाली जगह** → स्वचालित रूप से "आक्रामक" मोड सक्रिय हो जाएगा (कम डेटा प्रतिधारण अवधि, और कम सीमाएं)।

## विश्वास मॉडल।

क्लाउड गार्डियन केवल स्थानीय स्तर पर ही काम करता है। इसमें कोई नेटवर्क कनेक्टिविटी नहीं होती, न ही कोई डेटा संग्रह (टेलीमेट्री) होता है, और यह क्लाउड पर निर्भर नहीं है।

यहाँ जो लिखा है: `~/.claude/projects/` (लॉग फ़ाइलें, आकार, संशोधन की तिथियां), प्रक्रियाओं की सूची (सीपीयू, मेमोरी, अपटाइम, और क्लाउड से संबंधित प्रक्रियाओं के लिए हैंडल की संख्या, जो `pidusage` के माध्यम से प्राप्त की जाती है)।

**यह क्या लिखता है:** `~/.claude-guardian/` (state.json, budget.json, journal.jsonl, डॉक्टर बंडल)। सभी फाइलें उपयोगकर्ता के होम डायरेक्टरी में स्थित हैं।

**यह बंडलों में क्या जानकारी एकत्र करता है:** सिस्टम की जानकारी (ऑपरेटिंग सिस्टम, सीपीयू, मेमोरी, डिस्क), लॉग फ़ाइल के अंतिम 500 लाइन, प्रोसेस स्नैपशॉट, और गार्डियन का अपना जर्नल। इसमें कोई एपीआई कुंजी, टोकन, क्रेडेंशियल या उपयोगकर्ता की सामग्री शामिल नहीं है।

**खतरनाक कार्य - गार्डियन क्या नहीं करेगा:**
- किसी भी प्रोसेस को बंद करना या सिग्नल भेजना (कोई `SIGKILL` नहीं, कोई `SIGTERM` नहीं)
- क्लाउड कोड या किसी अन्य प्रोसेस को पुनः आरंभ करना
- फ़ाइलों को हटाना (रोटेशन = gzip, ट्रिमिंग = अंतिम N लाइनें रखना)
- नेटवर्क अनुरोध करना या किसी सर्वर को जानकारी भेजना
- विशेषाधिकार बढ़ाना या अन्य उपयोगकर्ताओं के डेटा तक पहुंचना

यदि किसी भी समय प्रोसेस को बंद करने या ऑटो-रीस्टार्ट की सुविधा जोड़ी जाती है, तो यह एक स्पष्ट 'ऑप्ट-इन' विकल्प के साथ होगी, जिसका दस्तावेज़ यहां दिया गया है, और डिफ़ॉल्ट रूप से यह बंद रहेगा।

## डिजाइन सिद्धांत

- **तथ्यों पर भावनाओं का ध्यान:** प्रत्येक कार्रवाई एक जर्नल प्रविष्टि लिखती है; क्रैश बंडल स्थिति को कैप्चर करते हैं, अनुमानों को नहीं।
- **निश्चित:** कोई मशीन लर्निंग नहीं, केवल फ़ाइल की उम्र और आकार के आधार पर हीuristic का उपयोग। एक निर्णय तालिका जिसे आप 60 सेकंड में पढ़ सकते हैं।
- **डिफ़ॉल्ट रूप से सुरक्षित:** रोटेशन = gzip (उलटने योग्य), ट्रिमिंग = अंतिम N लाइनें रखना (डेटा संरक्षित), v1 में कोई फ़ाइल डिलीट नहीं की जाती।
- **सरल निर्भरताएं:** commander, pidusage, archiver, @modelcontextprotocol/sdk। बस इतना ही।
